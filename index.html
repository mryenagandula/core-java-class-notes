  • To write an interface we have to use the “interface” keyword instead of class.

Exception Handling (Working with Exception Handling)

In every programming language we have 2 types of errors.

  1. Compile Time error
  2. Runtime Error

Compile Time Error

Question: What is the compile time error ?

Answer: The error which is detected by javac so that we can  not compile the file and there is no class file will be generated.

Question: When will we get a compile time error ?

Answer: Mostly will get by syntax mismatch like {} , ; : etc.

Question: What will be the solution for compile time error?

Answer: This javac is showing instructions like where we made mistakes and what required.So that as a developer we have to recheck and solve it.

Runtime Error

Question: What is the runtime error?

Answer: Which is generated by JVM (Java Virtual Machine) not javac. So that we can compile and we can get a class file and some time will execute, but some time it won’t be executed.

Question: When will we get a Runtime error

In 2 cases we get runtime error

1> Logical Problems

2> Memory Problems

Question: What will be the solution to handle runtime error.?

Answer: To handle runtime error, we have predefined statements, those are

  • try
  • catch
  • finally
  • throw
  • throws

try

Using this we can keep exception details to handle syntax.

try {

        //exception details

}

catch

Using this we can handle the try block of exception syntax.

catch (exception_name exp){

// client conversion messages

}

finally

Using this we can say exceptions are closed.

finally  {

        //exception closed statements

}

throw  

Using this we can raise exception

throw object_name;

throws  

Using this we can report exception details to JVM the JVM will handle the exception.

throws exception_name

 More Concentrating on Runtime Errors
 Logical Problems

Here logical problems means do not think logics are wrong. Then when clients are passing positive requirements to the logic, then it will work but when clients are passing negative requirements then it won’t work.

Memory Problem

This is what we can get due to memory problems like hard disk failure, stackoverflow, etc.

Working with Packages

Why Packages?

Easy Identification and easy recognition. So that we can use it easily.

What is the package?

Collection of related classes and interfaces etc.

We have two types of packages

  1. Predefined Packages for predefined classes
  2. User Defined classes for both predefined and user defined classes
